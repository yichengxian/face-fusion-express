# code对照表



## 通用

| code | 说明         |                                                              |
| ---- | ------------ | ------------------------------------------------------------ |
| 200  | 成功         | 返回成功                                                     |
| 500  | 失败         | 返回失败                                                     |
| 400  | 参数校验失败 | 请求参数未传                                                 |
| 404  | 接口不存在   | 请求方式有误或者请求路径不存在                               |
| 700  | 操作过于频繁 | 不要尝试同时发起两次请求(根据访问路径和ip进行访问限制(1s/次)) |



## 业务

业务代码code为是一共5位

以用户code为例：

| code  | 说明              |
| ----- | ----------------- |
| 10001 | 用户token校验失败 |

- 10 代表业务为用户业务
- 001 为该业务的中的唯一标识

组合后为10001

### 用户 

业务10

| code  | 说明               |                              |
| ----- | ------------------ | ---------------------------- |
| 10001 | 用户token校验失败  | token未传或者token失去完整性 |
| 10002 | 用户不存在或被禁用 | 用户不存在或被禁用           |
